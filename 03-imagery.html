Graphic Specifications

Image Standards

Image Standards

  • Cut all the same height and widths so they scale the same way
  • Don’t bleed transparent images over the left/right edges.
  • Coincides with TI's branding: open, clean, crisp
  • When dealing with a TI logo or sub-brand, do not use reflection or any other effect to alter the brand

Responsive Design

  • Images have been sized for responsive design, to grow & shrink as needed in it's container.
  • Border-to-border images:
    • Optimized for between 2-to-4 times the display size (the image size in a layout displayed in a 2560px wide browser window)
    • Using JPG format at low compression, useful for images that must fill the entire container
  • Transparent image specifications (object photos):
    • They have been optimized for 1.5 times the display size (the image size in a layout displayed in a 2560px wide browser window)
    • Using JPG format at low compression, useful for images that must fill the entire container
    • Using PNG-24 format for the alpha channel, useful for gradient backgrounds decided by CSS

Dos & Don'ts

NOTE: Styling image templates are available under Image Templates

1. For product shots, please remove the background from images.

Do not place a black line around the box of the image.

Do.

Crop the image so it is clean and open.

Do not.

Don't leave a black border and unneeded background on the image.

2. On gradient or coloured backgrounds, use a PNG-24 format for a product image on a transparent background.

Use a PNG-24 on an alpha background to preserve it's transparency. Use a PNG-24 on an alpha background to preserve it's transparency.

Do.

Use a PNG-24 on an alpha background to preserve it's transparency. This is useful for imagery that may be going on backgrounds decided by CSS, like stacks or feature boxes.

Do not.

Don't use a PNG-24 if the image is going on a white background. The PNG file-size is much bigger than JPG. For best results & to decrease load time, use a JPG.

3. Use the whole cover artwork to represent a PDF brochure or manual, not just a portion.

Do.

Use the template in Image Templates to properly feature the entire cover of the PDF.

Do not.

Don't use just a cropped detail of the PDF cover, free of a grey outline and shadow on the bottom.

4. Avoid presenting charts in perspective. Show charts straight on.

Angles make text difficult to read at smaller sizes.

Do.

Use a flat, straight-on view of diagrams and charts to increase legibility.

Do not.

Don't use a perspective or odd angle on any information-based graphics, it decreases the effectiveness.

5. Avoid complex imagery.

At most, use 2-3 elements: a brand, product & unifying background element.

Do.

Add a teal unifying background for multiple objects.

Do not.

Don't add more than 3 objects with off-brand elements in unnatural manner. Piling images on top of each (like in the example above) reduces their clarit & effectiveness.
Also, please don't add curved type on a skewed angle. You know better.

6. Avoid product shots at awkward angles.

Make the product shot looks stable.

Do.

Make sure the image is presented in a realistic, stable view.

Do not.

Do not put image on angle that looks forced or awkward.

7. Adjust light levels for improved visibility of details.

Do.

Adjust the levels in photoshop as needed to make the product shot more visually appealing & legible.

Do not.

Don't just leave the product shot as is. Connect with the client if nothing can be done to see if they have higher quality images.

8. Avoid campaign-based imagery.

For non-campaign pages/content,use imagery that can have longevity on the web and is independent of any specific campaign graphical styles, unless it is specifically campaign related.

Do.

Use clear, clean, high-reolution imagery that is not tied to to a campaign on non-campaign pages.

Do not.

Don't use campaign-based imagery on non-campaign pages. The content will date the page.

9. Don't use reflection on images that are presented on an angle (Isometric perspective).

Use a 40% drop shadow effect & light reflection, aligned evenly with the image.

Do.

Do use a slight reflection as needed, aligned evenly on the bottom of the product.

Do not.

Don't reflect the image on the tip of the image, like it's been dropped on the corner of the product.

10. Use quality images.

Any image on the web is a reflection of the TI brand.
If you can't find a quality image to represent your product, you may be better off not using any image at all until you can get a proper image.

Do.

Leave the image blank in place of adding a poor image that does not reflect TI's standards & expected level of quality. This can also free up load time.

Do not.

Do not use blurry, badly-cropped, low-quality imagery as it misrepresents the brand.

11. Represent chips realistically

Use straight up shots with reflection and drop shadow. Avoid angles whenever possible.

Do.

Have the product facing straight with a light reflection and drop shadow. This may not always be possible, depending on the product make & model.

Do not.

Don't put the image on a skewed, unnatural angle. As well, make to include the reflection & shadow.

Recreate Smaller Images In The New Dimensions

How to recreate (or “res up”) an existing image that is too small.

Step 1:

Open the photoshop template. Check to make sure the file dimensions are the image dimensions with no extra dead space.

Note

Since a line has a width, these kinds of indicators are imprecise in Photoshop (cut inside the line, or outside of it?). If you ever have to do this, it is recommended that you use a “slice” as well.

You can double-check if you know where the image resides on the site using right-click & “View image info” in Firefox.

So what size does this file need to be to ensure that the crop area will equal 960 pixels? Do I need to do a lot of fancy math?

No! (well not much) Mark the crop-line position with Photoshop guides.

Control-click on the crop-line layer in the Layers palette:

Check the dimensions in the Info panel (right side of photoshop screenshot):

Open the windows calculator and divide 960 by the width of the image (e.g., 274px). Copy the result:

Step 2: Back in photoshop…

Select Image > Image Size… and select “Percent” in the dropdown next to the the width in either the Pixel Dimensions width field:

Paste the number from the calculator tool into Width field, move decimal point two places, & hit “OK”

What can I use as-is and what do I have to rebuild?

  • Your crop-line indicator might be rasterized, so when you crop & export your graphic, rely on your Photoshop guides.
  • Raster images will have to be re-imported, resized, and repositioned.
  • Gradients will sometimes res-up well enough. But sometimes you will have to re-create them. Check the quality and use your best judgement.
  • Any layer effects should scale with the image. Check the quality.

Note

If the resolution of the original file from the vector smart object is over 72dpi the dimension of the smart object can be misleading. Check the resolution of the original, or if not available, just export the image and check the quality visually.

Step 4: Replace an original stock photo with effects applied to it

The old layer will have become fuzzy in the resize. To resolve this:

  • Import the image as a smart layer
  • Do not link the smart layer, embed it so you don’t lose it if someone moves the original.
  • Make the smart layer semi-transparent
  • Scale it & reposition  over the old art layer
  • Once the art is aligned you can copy/paste the effects to the new layer: I left off here – how to preserve masks, etc

If you have a layer mask to redraw, try merging a rasterized duplicate of the new photo layer over the old one.

  • Erase the pixels on the old masked layer first, leaving the mask preserved on a clean layer
  • Then right-click the new image layer to merge it down into the layer containing the mask
  • Select “Preserve” when prompted – do not “apply” or you’ll lose the ability to edit the mask.

  • If the mask is too fuzzy, try sharpening it (select the mask icon in the layer palette first).
  • If that doesn’t work, you’ll have to redraw your mask.

Finding the original source file

The original image should have an IMS request number in front of the filename – search \\4com.itg.ti.com\ims\ for that number and you’re likely to find the asset.

  • If that doesn’t work, search the IMS ticket SharePoint site for that ticket and see if it’s attached to the ticket.
  • If not, send an email to the UI team to see if anyone recognizes it and hopefully someone will know where to look.

Naming Convention

All graphics created for JIRA based requests/stories, the following naming convention is to be used:

[ JIRA project ID ][ Issue/Story ID number ]_[ Description ]_{ Region code }.[ Extension ]

JIRA project ID
Every JIRA project has a unique identifier. Use this identifier (lowercase) for your images but remove the "DC_" prefix. It will help us track the original graphics and Jira ticket.
Issue/Story ID number
Every issue or ticket or story has a unique number.
Description
Be descriptive of the image (limit to 20 characters and use underscores for word separators.
Region code
The region code is only required for Banners.
Extension
3 character extension, for example "png", "jpg", "gif", "svg"

Examples:

  • ticompat52_16611_wcSmartWaterMeterSBD.jpg
  • dm3637_Industrial_Ethernet_lg_en.jpg

FAQs

Q: When a change to an existing image is requested what do I do with that asset?

A: Retain the original image identifier in the filename. This will ensure the change is site wide where ever that asset is being used.

Q: What if there is a change to an existing image but that change only needs to take place on a specific page?

A: Use the following Steps for creating derivative work while maintaining original nomenclature:

  1. First locate original artwork in older request project directory
  1. Duplicate the original artwork but retain filename etc
  1. Adjust filename slightly by retaining the original file name and adding a date at the end of the file name. For example: dm16611_wcSmartWaterMeterSBD_en_2012-02-14.ai. This will ensure the original asset is still traceable.
  1. Once edit is complete and new web-based graphic is saved out, keep the derivative artwork source file in the old project folder
  1. Create a shortcut in your new project folder that points to the folder that contains the original and derivative artwork pieces. This helps locate the latest source artwork for a graphic even though the filename has not changed.

Q: If I find the original stock image in the directory, can I reuse it to create a new graphic?

A: NO. Do not reuse any stored stock imagery as this may be in violation of our contract, unless it is part of our Digital Asset Management system.

Q: What if I cannot locate the original artwork?

A: Some assets are older than this naming system. Here are some tips for finding lost artwork.

  • Send the image to the larger UI Design group to see if anyone can recognize it.
  • Ask the requestor for the original artwork.
  • Out of luck? You'll just have to create a new image.

Ticom C 1 To C 6 Semi Transparent

PNG-24 can be challenging to export to an acceptable file size, but is required for objects with shadows or background shading over a transparent background. To reduce file weight, reduce the size of the image.

Be sure to use the dimensions for all images in a set to avoid uneven scaling on large screens.

Dimensions for semi-transparent images are smaller than JPG or PNG-8

  • PNG-24
    • Under 60 KB
  • Standard width - 360 px
  • Min height - 135 px (preferred)
  • Max height - 180 px high
    • Since PNG-24 files are very large, test file size and how the images scale in large/HD screen size before exceeding the max recommended height.
    • All images in a set should be sized the same way to avoid component assymmetry (e.g., disproportional scaling in sub-family boxes at higher breakpoints).

Typically a clipped photographic object over a masked pattern, shadow, or glow effect.
