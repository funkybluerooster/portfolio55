Overview

Design elements are a collection of recurring singular parts that appear multiple times across a site that makes up a pattern library. They can be combined together to create larger patterns, like modules. In this section you will find elements like buttons, form elements, icons, layouts, message boxes and typographic styles. These elements will aid you in consistency with the user experience, reusability and maintability of the site.

Buttons

Buttons for forms or functions

Unlike CTA buttons and links, Form/Function Buttons are specifically associated with a form or a tool or a function rather than linking to a page.

When to use input vs anchor link vs button

The question of which to can be answered following the basic rules of semantics, which is using markup that reflects the meaning of the content. Using semantic HTML makes your content explicit, which gives you better browser compatibility, accessibility and SEO.

<a>

Should be used when you are navigating the user to a page or resource.

<input>

Used when submitting or resetting a form. If you need to track use of an input button, make sure the script it activates contains a linkID.

<button>

Is supported in HTML 5 only and is semantically appropriate in either case.

Don't forget, a, input and button are inline elements by default and you may need to wrap them in a div.row or a p as they have no spacing on their own.

07 Download Buttons

Best usability/user experience practice is to set expectations for our customers

If the link is not a direct download, it is best then to use a general CTA link. Otherwise it can cause a negative user experience if a customer expects a download but is instead sent to another page where the download is not obvious.

For file size notations on PDFs or other downloads:

  • “b” stands for bit,
  • “B” stands for byte,
  • megabytes is “MB”,
  • kilobytes is “KB”
  • and gigabytes is “GB”

If the file will be downloaded from this link, make sure to include the type and file size, so our customers can anticipate what will happen next.

This is especially important for those who have to watch data download costs or are limited in bandwidth (global audience).

Circle Number Instructions

Step circles (or numbered steps) helps identify and illustrate steps in a process or instructions based on the content defined by the content. Can also be called 'Numbered Steps, 'Circle Number and 'Progress Steps' are meant to provide a simple way of illustrating steps in a process without the use of images. These step circles can be configured in anyway you'd like, however, the final step should use the check mark graphic.

Why this solution?

The user needs to know where they are in the process or instruction (such as install or set-up).

When to use pattern

Use Step circles in a predefined process that the user may use to complete a process or to explain the flow of content. Do not use for routine tasks for which steps certain steps are expected.

What is the solution

Show Step circles (or numbered circles), which is a persistent navigation circle displaying a number of steps and highlighting the current step and visually what is left to complete process.

  • Step circles should be persistent throughout the process.
  • Use short names for steps.
  • Ensure the Step circles are accurate in all cases.
  • No user should be able to skip steps or encounter steps that aren't reflected in the Step circles.
  • Completed step circles should have a teal background to show completetion.
  • Active or current Step circles should have a dark gray background.
  • The completed step or last step should be disabled or grayed out until completion.
  • Ensure the design can't be mistaken for clickable navigation.
  • There should be no more than 10 steps.

The term 'step circle' can be applied to instructions showing a dynamically updated view of progress (as in the case page with step circle being used).

    Message Boxes

    Alert messaging can take many forms, it can be informational ("nice-to-have"), error related ("critical"), confirmational ("success/denial messages").

    As such, we have included sub styles that control colour, icon, etc., and three main types of messages: Static, Dismissable, and Interactive.

    Static messages are on the screen all the time, and cannot be removed. An example would be an alert (.call-out-box .alert), which could be used for a "click here to register!" area on a form.

    A dismissable alert would be present on page load, but would contain a close button (x) through which the user could dismiss the message. The only way to bring the message back would be through reloading the page.

    Interactive alerts are triggered open by clicking an icon or text, and can then be triggered closed by clicking that same icon or text, or a close button (x) present in the alert container. These are most commonly referred to as "info panes" and are triggered through the use of an "i" icon by default.

    • occurs once only on a page
    • often located underneath the tabbed navigation
