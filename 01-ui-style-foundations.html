02 Multi Screen Design Principles

Overview

We generally design TI.com with desktop users as its primary audience (full site). Since Tablets are increasingly being used as working computers, have larger screen sizes and higher resolutions, they will display the same full site as the desktop users. All new patterns created for TI.com will be touch friendly to work with both desktop and tablet touch screens.

Users coming to TI.com from a mobile device will see an adaptive view (mobile site). This view will be uniquely curated and will contain only the most important parts of TI.com in the near term. If a portion of TI.com needs to address mobile users, it will be accommodated on project-to-project basis. We may have some cases where we incorporate a responsive layout (i.e. community).

November 2016 update:

  • We are transitioning the Smartphone site (m.TI.com) to be part of the core UI virtual team. It will be adaptive not responsive and uniquely curated. Not everything on TI.com desktop will be part of the smartphone offering. A curated experience will be available for the mobile devices.
  • Planning to restart the TI mobile app development in Q3, 2017 and the XA team has been asked to do further research before committing to redesigning it. This team will be transitioned to the new applications virtual team that is responsible for internal apps.

Metrics

According to data gathered in Q1 and Q2 of 2015, TI.com metrics show the majority of tablets and desktops using screens sized at > 1200 pixel width (43% have their browser width between 1200 and 1400). These metrics also show the majority of smartphone users have screen sizes in the 5' range (Top 10 devices are between 4.7 and 5.5), therefore only one breakpoint is needed between smartphones and all other devices. Tablets and desktops should use the same full site view.

Recommendations:

  1. Create two views for TI.com: 1) full site and 2) mobile site
  2. Both views will account for touch screens and multiple devices including smartphones, tablets and desktops
  3. Tablet and desktop will use the 'full site' view by default
  4. Smartphones will use the 'mobile site' view by default
  5. Full site will not use content choreography (slide and stack)
  6. In the near-term, the mobile site will use content choreography and content curation (limited content) and focus only on high value pages
  7. Customers can switch between views on any device, but mobile view and full site view will not be a one-to-one match in the near-term
  8. Only one break point will be used (called the 'mobile adaption point')
  9. Full site and mobile site will be driven from one responsive and adaptive code base (no 'm.site')
  10. Over time, we will change our strategy based on a 'spectrum' of all known devices that engineers use to interact with TI.com (future workplace devices could appear, such as VR headsets and Google Glass)

Reference: Multi device strategy working document in the XA shared document repository. (if you get an error pop-up, you can still open directly into power point)

03 Touch Friendly Design

In general

Apple's iPhone Human Interface Guidelines recommends a minimum target size of 44 pixels wide 44 pixels tall, which is what we use at TI. We also follow recommendations on Google’s Material Design. Designers and developers must fine tune each new interface according to human factors details.

These details include creating designs that offer:

  • Direct manipulation through rotation or gestures
  • Visual metaphors work well like toggle switches, sliders, flicking pages
  • Consistency: incorporates features and behaviors in ways people expect
  • Feedback to acknowledge actions
  • Users control apps. Apps do not make decisions for users. Help people feel in control
  • First impressions: your icon has to be your business card. Polished and clear. And the first time the app launches it must work correctly, simply and clearly. It should not require a set of instructions and complex help text.
  • Target area is universally 44px square, and please be aware of multiple button placements relative to one another.

Details

Some additional reading can be done here. In this article, Jonathan Stark at Creative Bloq outlines a lot of mobile design principles such as:

  • Keep a mobile mindset – it's different than desktop
  • Remember that mobile contexts are on-the-go. Keep in mind the very meaning of the word mobile
  • Design for thumbs as well as fingers
  • Controls should go at the bottom of the screen (beneath the content)
  • Scrolling: avoid it when you can
  • Do not use rollovers as triggers -- you must be able to click or tap to activate
  • If a keyboard is required: make the layout appropriate to integrate that
  • The orientation should not suddenly change. Portrait is most popular but especially if there is typing, landscape should be allowed
  • When launching, resume where you left off.

Navigation models:

  • Single screen utility apps (eg Weather app on iPhone)
  • Tab bar: Three to six distinct content areas (eg Twitter for iPhone)
  • Drill down: List and detail content hierarchy (eg Settings app on iPhone)

Gestures

  • They should not be too hard to find... make them discoverable. Example: on the promotional iPads mounted in Apple’s retail stores. When a page first loads, any scrollable areas do a quick 'reverse scroll' into their default position.
  • Two hands vs one hand. Android vs mac pinch and zoom.
  • Gestures are mostly 'nice to have' features, not everyone will notice or appreciate them

Communications:

  • Provide feedback for every interaction
  • Modal alerts should be limited try using confirmations instead: “Are you sure you want to delete”

05 Typography Fundamentals

Brand Fonts

Brand fonts

Helvetica Neue

Our corporate typeface is Helvetica Neue, which is simple, clean and precise. The Helvetica Neue family of fonts is versatile and contemporary, allowing flexibility across many applications.

When setting type for different applications, use Helvetica Neue 45 Light for entity names on communication pieces. Helvetica Neue 55 Roman is the primary typeface for body copy, while Helvetica Neue 75 Bold is for headlines.

Installation

Font files are located on 4com:  \4com\ims\Templates\Fonts\   When installing fonts:

  • Install these fonts by pasting them into  C:\Windows\Fonts\  
  • Restart applications after fonts have been installed

Also please refer to TI Brand Guide for more information.

Line Length

Best practices

Copy length is a legibility issue. When your line of copy is too long, your eye tires and it makes it hard to read.

Aim to keep your line length around 45-75 characters, with an optimal length of around 66 (this includes spaces).

Our CSS limits a <p> line length to 60em.

Want to learn more?

The quinessential read on typography: Robert Bringhurst's The Elements of Typographic Style (web version: http://webtypography.net/)

06 Layout

The Grid

The grid we are using is based on gridiculo.us.

Key Features
  • 12 col grid system
  • Allows for nested grids
  • Responsive
Basic structures
  • Each row is defined using class="row"
  • Each column is defined by using class="c{size of column}" where {size of column}=1, 2, 3, … 12.
Why 12?

12 is a common grid system because it can divide in different ways. For example:

12 = 6 + 6 = a 2 column layout

12 = 4 + 4 + 4 = a 3 column layout

12 = 3 + 3 + 3 + 3 = a 4 column layout

12 = 8 + 4 = a main content area with side bar

etc.
