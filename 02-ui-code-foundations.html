01 Ui Design Checklist

This is a guideline for all designers to follow to ensure proper use of our established methods and approaches to help minimize any inconsistencies, and style conflicts that can affect the overall display of a given page.

  • UI Design component testing steps:
    • Follow naming convention guidelines
    • Check your CSS class names
      • Download "check_your_css_class_names" file , paste your HTML code in the and save it locally. Then open it in a browser and check in the inspector for any conflict with new classnames that you intend to add to the SASS repo. If the new names are not present in any of the below files, you can proceed to request a merge into the main SASS repo. If there are duplicates, please go back to your CSS/SASS and choose a new [unique] name.
    • Operating systems – When testing, you should keep in mind that browsers in different OS may render differently (e.g. MS Chrome v OSX Chrome). These are the supported systems to check on:
      • Windows (Win10 and Win7)
      • Chrome
      • Mac X
    • Browsers – These are the most used that should be checked on for any cross-compatibility issues:
      • Firefox
      • Chrome
      • IE 11, IE Edge
      • IE 8 ( Optional as this is going through graceful degradation)
      • Safari
    • Devices
      • Desktop
      • Tablets (portrait, landscape)
      • Smartphone (portrait, landscape)
  • UI Design component/pattern reusability:
    • Modular
      • Can this be reused in other existing components
      • No class name conflicts will occur in other uses
    • Semantic (SEO friendly)
      • Review semantic hierarchy (e.g. h1 vs h3 in component)
      • HTML5 tags (e.g. nav in components vs nav in page) Additional UI design test: Check CSS naming convention:
      • Verify mapping of z-index
