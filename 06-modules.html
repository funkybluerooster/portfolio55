Feature Boxes

Feature Boxes are used to highlight content on a page. They are declared by including the class featurebox in your parent row declaration.

The featurebox class ensures that you get a certain look to the box but the inside of the box can be arranged in different ways as it is based on nesting the grid.

This style should not be overused on a page otherwise it loses it's purpose – make sure there is visual hierarchy to the content.

Remember to avoid double right padding if you are nesting a row inside a column. See nested grids for more info on cEnd class.

For consistency, ONLY use the following button labels:

  • More = Learn more, more info, etc.
  • Order now = Buy online, buy at TI, but, buy today etc. "Order" is company standard messaging.
  • Download = Make sure the size and type is above the button. *
  • Get software

*If the Download is NOT in a feature box, then use the download button style.

Forms

Basic structure and layout guidelines

All the form elements have been defined to function look a certain way via the css. Please do not change how the forms look or function.

The layout and style of forms have been determine with the collaboration of the XA team.

A responsive form template is available to get you started

Alignment of elements

Optional vs Required fields

Forms have fields that are either required or that are optional. How these are presented may vary depending on how many of each. We have distilled this down to two scenarios to help make the differentiation obvious to customers and make the form very scanable.

  1. A few optional fields (less than 1/3rd of fields are optional)
  2. Many optional fields

A few optional fields

When you have just a few fields that are optional, use the optional text indicator with the label. This text works best when it is with the label as it will work well with any form element (ex. drop downs) and is clear from other things like error messages or tool tips.


When you are dealing with labels inside the input field, use "optional" in brackets after the label.

Require fields

Use the asterisks for required fields for forms where there are required fields to fill in.

Copyright Information

When you need to use a copyright notice, it should usually have a date and the company name.

About the date

The date is usually the first year that the content was published.

For example if we published something in 2008, our copyright notice says "© Copyright 2008 Texas Instruments Incorporated"

Updating the date (date ranges)

Sometimes you'll see a hyphen in the date. This happens when an update is made to the content, but in a different year than when the content was first published. For example:
"© Copyright 1995-2008 Texas Instruments Incorporated" This means that we first published our content on the web in 1995, and the last update made was in 2008. If we make an update to the site in 2009, then we should reflect this in the copyright notice.

So going forward, always keep the original date of publication. You just need to add or update the date after the hyphen if we're in a new year.

The loading icon can be used as a placeholder to indicate that a specific element is loading

  • Required when server response time is not instantaneous, and will take an indefinite amount of time to load
  • Icon takes on the maximum height/width of the object that is being loaded up to a maximimum icon size of 30px x 30px (without causing distortion)
  • No messaging/text is used - following google's material design guidelines
  • Standard TI animated icon should always be used
  • Spinner should be centered within masked area horizontally and vertically
  • Spinner is replaced with the loaded element when loading is complete

Pages

The spinner icon can be used to indicate that an entire page or large section of a page is loading.

  • required when server response time is not instantaneous, and will take an indefinite amount of time to load
  • A white (60% opacity) mask should be used over the area that is loading (eg., results area when filtering)
  • no messaging/text is used - following google's material design guidelines
  • standard TI animated icon should always be used
  • spinner should be centered within masked area horizontally
  • spinner-container has a top margin of 300px
  • animation should be automatically removed once results are retrieved
  • hidden class is required to hide the animation before it is required/triggered by JS/JQuery (unique to each usage)

NOTE: The "spinner-target" class should be applied to the div that is to be masked out by the loading icon. This class can be appended to an existing DIV's class. (ie., class="main-div-class spinner-target")

Modal Windows

A modal window is a secondary window that opens on top of the main one. Users have to interact with it before they can carry out their task and return to the main window.

You should use modal windows when there are steps the user needs to do before the task can be completed. Using a modal window instead of a full page allows users to maintain the context of their task.

Include a clear, visible title that matches the clicked button

When a modal window opens, it’s important for users to know where it’s coming from. Your modal window should include a title that matches the button they clicked to open it. This shows consistent behavior and tells them what task they’re doing.

Modal Windows

Modal windows have the following characteristics

  • background set to .6 opacity black in browsers
  • Rounded corner: .2em
  • box-shadow: 0 0 10px #000

Show And Hide Content

Accordions

An accordion is used to show (and hide) content that is broken into sections. Accordions make a long page shorter by hiding content blocks, while at the same time the section headers naturally float higher up the page allowing the user to scan sections topics easier.

NOTE: Browser find in page function (when you use [CTRL]/[COMMAND] F on keyboard) is hindered by the accordion if the search string is located within a hidden content block.

Accordion structure

An accordion set has a container with .collapsible .js-collapsible classes. Each item contains the classes .collapsible-item .js-collapsible-item. Each item has a trigger with the classes collapsible-item-trigger js-collapsible-trigger applied to a header tag and the collapsible content with the classes collapsible-item-content js-collapsible-content.

Items expand one at a time inside a set, so to allow multiple items to remain open, use multiple containers each with one item.

All .js- prefix classes exsist to seperate the javascript triggers from the style.

.js-collapsible-anchor class can be added to the container to enable anchor scrolling to collapsible content on trigger.

.collapsible-item can be given id names to allow for anchor linking to expanded content. See example on following buttons:

Link to accordion A Link to accordion B Link to accordion C

Javascript (uses Jquery)

A page with accordions requires the following javascript after the accordions:

$(function(){
						  var SLIDE_DURATION = 300;
						  $('.js-collapsible-item').each(function(index, element){
						    var self     = element,
						        $this    = $(element),
						        $trigger = $this.children('.js-collapsible-trigger');
						
						    $trigger.click(function(e) {
						      $this.parent().children('.js-collapsible-item').each(function(index, element){
						        if (element == self) {
						          $(element).toggleClass('is-expanded');
						          $(element).children('.js-collapsible-content').slideToggle({ duration:SLIDE_DURATION });
						        } else {
						          $(element).removeClass('is-expanded');
						          $(element).children('.js-collapsible-content').slideUp({ duration:SLIDE_DURATION });
						        }
						      });
						      if($this.parent().hasClass('js-collapsible-anchor')) {
						        setTimeout(function(){
						          $('html, body').animate({ scrollTop: $(element).offset().top }, 'fast');
						        }, 400);
						      }
						    });
						  });
						
						  // start required script for scroll to and open expandible with anchor
						  var collapsibleIsVisible = $('.js-collapsible-item').length;
						  if(collapsibleIsVisible) {
						    var pageUrl = location.href;
						    var ActiveElementName = pageUrl.split('#')[1];
						    if(ActiveElementName) {
						      autoOpenCollapsible('#' + ActiveElementName)
						    }
						    $('a, button').click(function() {
						      var optionId = ($(this).attr('href'))
						      autoOpenCollapsible(optionId)
						    })
						  }
						  function autoOpenCollapsible(option) {
						    var $parent = $(option).parent();
						    $parent.find('.js-collapsible-item').removeClass('is-expanded');
						    $parent.find('.js-collapsible-content').hide();
						    $(option).addClass('is-expanded');
						    $(option).children().show();
						  }
						  // end required script for scroll to and open expandible with anchor
						});

Area commented // start required script for scroll to and open expandible with anchor is optional and required only for enabling on-page anchor link to expandable area and automatic open.

				$trigger = $this.children('.js-collapsible-trigger');

					if (element == self) {
						$(element).toggleClass('is-expanded');
						$(element).children('.js-collapsible-content').slideToggle({
							duration: SLIDE_DURATION

							duration: SLIDE_DURATION
						});
					}
				});
				if ($this.parent().hasClass('js-collapsible-anchor')) {
					setTimeout(function() {
						$('html, body').animate({
							scrollTop: $(element).offset().top
						}, 'fast');
					}, 400);
				}
			});
		});

		// start required script for scroll to and open expandible with anchor
		var collapsibleIsVisible = $('.js-collapsible-item').length;
		if (collapsibleIsVisible) {
			var pageUrl = location.href;
			var ActiveElementName = pageUrl.split('#')[1];

03 Sortable Tables

To create a sorting table you need to

  1. include the call to the jquery script
  2. inline script for setting the default column to sort first

IMPORTANTOn non-LiveSite pages, Do not put jquery.tablesorter.min.js in the head, it doesn't work there. However in LiveSite, it will function fine if placed in the scripts area of the content component.

To add another sortable table in the campaign page

Copy/paste an existing table, and assign it a unique id such as tblSort3. tblSort3 through tblSort8 are included in the template so you don't have to remember to add them to the inline script.

Tips

  • Don't remove the table header or table body tags. They are essential for the table to function.
  • There is no zebra striping or rollover behavior.

Changing the default column to sort and sorting order on a sortable table

Each table has a bit of inline script code that determines which column is sorted on page load. This example sets the first column as the default:

$("#tblSort1").tablesorter( {sortList: [[0,0]]} );

  • The first digit in the yellow hilighted area specifies the column to sort by default.

    • Note that zero represents the first column (whether its hidden or not)
    • 1 represents the second column, etc.

  • The 2nd digit directs the table to sort ascending or descending.

  • Zero specifies ascending (by A-to-Z or by smallest to greatest number).

  • Change zero to 1 for descending order instead.

Turning off sorting for a column

headers: { x: {sorter:false} }

  • Where "x" is the column number
  • Remember that columns start numbering at 0
  • If you need multiple columns separate items by comma. For example: headers: { 2: {sorter:false}, 3: {sorter:false} }

04 Sticky Tables

What are sticky table headers good for? Well, let's say you want to display a long list of fairly uniform tabular data, like product listings or relevant statistics. But you don't want your users to get lost in the data as they scroll down on the page. Sticky Table Headers to the rescue: By applying the StickyTableHeaders jQuery plugin to the table, the column headers will stick to the top of the viewport as you scroll down.

It is recommended not to alter the scroll position when using sorting mid-scroll. This may cause user to manually scroll where the intention is to observe starting values.

If the client is sensitive to highest and lowest values, they should just consider a standard table in place of a sticky nav since highest/lowest values are clearly displayed.

Requirement

  • Jquery Plugin

Do's and Don'ts for sticky table header

Do use when:

  • When the data in the table requires the columns to be in view to distiguish similar data.
  • When you need to know the column headings to interpret the data

Don't use when:

  • Column data is obvious.
  • It does not help the customer understand the data.
  • Only for bells and whistles, superfluous.

Technical Documents

Technical Documents

On product portal pages the technical documents tab often features a list of technical documents.

There are 2 different ways to display this content depending on how many featured technical documents you have:

  1. 6 or less featured documents
  2. more than 6 featured documents

6 or less featured documents

  • Use a 2 column grid
  • Follow pdf cover image standards (see wiki for details and templates)
  • Link should be only the basic link style
  • Include format and size information

More than 6 featured documents

  • Use a 8-4 column grid
  • Create a spread of pdf covers use the pdf cover image standards (see wiki for details and templates)
  • Use an ul class="pdf" listing with simple link style
  • Include format and size information

See section on Lists with icons for code details

Download information format

  • Make sure to include the format and file size information of documents as some of these files can be quite large. (It's important to set expectations for our customers when they download.)
  • Only link the download text, not the PDF and document size info.
  • DON'T use buttons unless it is the ONLY download on the page.

See the Portal Family Template document for more details

Rss All Developer Details

Overview

This code pulls and displays a bulleted list of an RSS feed from E2E.ti.com blogs or forums or by using tags.

How do I know which blog, forum or tag to program?

For forums and blogs, the values for 'forum' and 'blog' are the trailing ends of the e2e.ti.com URL of the forum or blog.

For tags, it is the trailing ends of the e2e.ti.com URL query that a client should provide: http://e2e.ti.com/search?q=buck-boost&sort=date%20desc&category=blog&group=145&tag=buck-boost

blog

'Behind the Wheel' : http://e2e.ti.com/blogs_/b/behind_the_wheel, blog feed code is just the last entry 'behind_the_wheel' (for blogs use only the final blog name in the string)

forum

'C5000 Ultra Low Power DSP Forum' : http://e2e.ti.com/support/dsp/c5000/
forum feed code is the last 2 entries 'dsp/c5000' (for forums use the family and the folder names)

tags

Diversity Fuels Innovation, DFI and buck-boost : no spaces between tags, comma separated, no hyphens. ["dfi","diversity fuels innovation","buck_2D00_boost"] If a client wants a curated list to display, they can create a unique tag and apply it to the posts they want to appear.

The tags are the trailing ends of the e2e.ti.com URL query that a client should provide: http://e2e.ti.com/search?q=buck-boost&sort=date%20desc&category=blog&group=145&tag=buck-boost = buck-boost or http://e2e.ti.com/tags/Diversity%2bFuels%2bInnovation = Diversity Fuels Innovation

The code will not recognize a tag with a hypen (ie: "buck-boost"), so please use this instead of the hyphen: _ 2D00 _ (ie: "buck_2D00_boost")

NOTE: we currently cannot pull images into the feed.

The template is an html-formatted string, but must include specific tokens to display information properly. These are here as examples of what is displayed in the string. You can have as many of these in your string as you like.

date post was published

            Widget Slideshow Generator

            The TI slideshow generator is a utility that allows for the creation of image slideshows that can be used on TI.com. It allows the user to upload 2 or more images that can then be displayed - using the embed code the tool generates - as an image rotator or slideshow.

            Please note that the scripts associated with the slideshow have already been included in the header of ti.com, html template pages, and the microsite template.

            When to use:

            • Great for holding a lot of info in one compact space, so if you need to keep things together in one place, it’s a nice way of managing it
            • Message is very marketing driven and not imperative
            • Campaign based
            • Multiple diagrams or messaging
            Example:

            When NOT to use:

            • Banners – banners are administered by the business marketing groups via an admin tool. A playlist should be created and the only thing you need to insert is a playlist embed code with the playlist number.
            • When another solution is viable – REMEMBER: large images require load, so if there is a better solution, use it.
            • No SEO required (uses graphics)
